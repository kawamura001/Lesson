<span id="ローカルリポジトリの作成" class="fragment"></span><a href="#%E3%83%AD%E3%83%BC%E3%82%AB%E3%83%AB%E3%83%AA%E3%83%9D%E3%82%B8%E3%83%88%E3%83%AA%E3%81%AE%E4%BD%9C%E6%88%90"><i class="fa fa-link"></i></a>ローカルリポジトリの作成</h3>

<p>初期化して、現在あるファイルを追加して、コミットすればOK<br><br>
ファイルがなければ<code>git init</code>のみでOK</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git init
git add *
git commit -m "initial commit"
</pre></div></div>

<h3>
<span id="リモートリポジトリからプロジェクトをコピー" class="fragment"></span><a href="#%E3%83%AA%E3%83%A2%E3%83%BC%E3%83%88%E3%83%AA%E3%83%9D%E3%82%B8%E3%83%88%E3%83%AA%E3%81%8B%E3%82%89%E3%83%97%E3%83%AD%E3%82%B8%E3%82%A7%E3%82%AF%E3%83%88%E3%82%92%E3%82%B3%E3%83%94%E3%83%BC"><i class="fa fa-link"></i></a>リモートリポジトリからプロジェクトをコピー</h3>

<p>ターミナルでローカルリポジトリに移動して以下のコマンド</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>cd [ローカルリポジトリのパス]
git clone [リモートリポジトリパス] (例： https://github.com/jquery/jquery.git)
</pre></div></div>

<hr>

<h3>
<span id="ファイル更新までの基本手順" class="fragment"></span><a href="#%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E6%9B%B4%E6%96%B0%E3%81%BE%E3%81%A7%E3%81%AE%E5%9F%BA%E6%9C%AC%E6%89%8B%E9%A0%86"><i class="fa fa-link"></i></a>ファイル更新までの基本手順</h3>

<p>ざっくりは以下の様な流れ   </p>

<ul>
<li>ファイルを追加<br>
</li>
<li>ファイルをコミット<br>
</li>
<li>ファイルを更新</li>
</ul>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git add [ファイル名] //追加
git commit -a -m "任意のコメント"  //コミット (-aオプションは変更を自動検出してくれる)
git push origin master  //masterを更新
</pre></div></div>

<hr>

<h3>
<span id="git-addの使用例" class="fragment"></span><a href="#git-add%E3%81%AE%E4%BD%BF%E7%94%A8%E4%BE%8B"><i class="fa fa-link"></i></a>git addの使用例</h3>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git add . //すべてのファイル・ディレクトリ
git add *.css //すべてのCSSファイル
git add -n //追加されるファイルを調べる
git add -u //変更されたファイルを追加する
git rm --cached //addしてしまったファイルを除外
</pre></div></div>

<hr>

<h3>
<span id="git-commitの使用例" class="fragment"></span><a href="#git-commit%E3%81%AE%E4%BD%BF%E7%94%A8%E4%BE%8B"><i class="fa fa-link"></i></a>git commitの使用例</h3>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git commit -a //変更のあったファイルすべて
git commit --amend //直前のコミットを取り消す
git commit -v //変更点を表示してコミット
</pre></div></div>

<h3>
<span id="コミットの取り消し" class="fragment"></span><a href="#%E3%82%B3%E3%83%9F%E3%83%83%E3%83%88%E3%81%AE%E5%8F%96%E3%82%8A%E6%B6%88%E3%81%97"><i class="fa fa-link"></i></a>コミットの取り消し</h3>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git reset --soft HEAD~2 // 最新のコミットから2件分をワークディレクトリの内容を保持し取り消す
git reset --hard HEAD~2 // 最新のコミットから2件分のワークディレクトリの内容とコミットを取り消す
</pre></div></div>

<hr>

<h3>
<span id="コミットメッセージの修正" class="fragment"></span><a href="#%E3%82%B3%E3%83%9F%E3%83%83%E3%83%88%E3%83%A1%E3%83%83%E3%82%BB%E3%83%BC%E3%82%B8%E3%81%AE%E4%BF%AE%E6%AD%A3"><i class="fa fa-link"></i></a>コミットメッセージの修正</h3>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git rebase -i HEAD~2 // HEADから2件のコミットメッセージ
</pre></div></div>

<p>上記のコマンドを実行するとVimが起動し、最新から過去2件のコミットが表示されます。<br>
※ Vimのコマンドはこちらを参考に → <a href="http://qiita.com/merrill/items/9c800030333ab4c9408f">Vimコマンドまとめ</a></p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>pick {commit_id} {commit_meessage} // 2件目
pick {commit_id} {commit_meessage} // 1件目(最新コミット)
</pre></div></div>

<p><code>pick</code>の部分を<code>edit</code>もしくは<code>e</code>に変更後ファイルを保存し、<br>
修正が完了したら<code>--amend</code>オプションを付けてコミットする。</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git commit --amend
</pre></div></div>

<p>最後に下記のコマンドを実行し完了。</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git rebase --continue
</pre></div></div>

<hr>

<h3>
<span id="ブランチの作成移動削除変更一覧" class="fragment"></span><a href="#%E3%83%96%E3%83%A9%E3%83%B3%E3%83%81%E3%81%AE%E4%BD%9C%E6%88%90%E7%A7%BB%E5%8B%95%E5%89%8A%E9%99%A4%E5%A4%89%E6%9B%B4%E4%B8%80%E8%A6%A7"><i class="fa fa-link"></i></a>ブランチの作成/移動/削除/変更/一覧/</h3>

<p>ブランチは変更履歴を記録できる。</p>

<blockquote>
<p><a href="https://camo.qiitausercontent.com/0b981e128ae83c0eb684ba85501213f2f79354ce/687474703a2f2f7777772e6261636b6c6f672e6a702f6769742d67756964652f696d672f706f73742f7374657075702f636170747572655f737465707570315f315f312e706e67" target="_blank" rel="nofollow noopener"><img src="https://camo.qiitausercontent.com/0b981e128ae83c0eb684ba85501213f2f79354ce/687474703a2f2f7777772e6261636b6c6f672e6a702f6769742d67756964652f696d672f706f73742f7374657075702f636170747572655f737465707570315f315f312e706e67" alt="ブランチとは" data-canonical-src="http://www.backlog.jp/git-guide/img/post/stepup/capture_stepup1_1_1.png"></a><br><br>
<a href="http://www.backlog.jp/git-guide/stepup/stepup1_1.html" rel="nofollow noopener" target="_blank">ブランチとは? サルでも分かるGit入門</a></p>
</blockquote>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git branch [branch_name]  //ブランチの作成
git checkout [branch_name]  //ブランチの移動
git branch -d [branch_name]  //ブランチの削除
git branch -m [branch_name]  //現在のブランチ名の変更
git branch // ローカルブランチの一覧
git branch -a //リモートとローカルのブランチの一覧
git branch -r //リモートブランチの一覧
git checkout -b branch_name origin/branch_name //リモートブランチへチェックアウト
</pre></div></div>

<hr>

<h3>
<span id="編集をマージ" class="fragment"></span><a href="#%E7%B7%A8%E9%9B%86%E3%82%92%E3%83%9E%E3%83%BC%E3%82%B8"><i class="fa fa-link"></i></a>編集をマージ</h3>

<p>master以外のブランチで編集した箇所をmasterに反映させる</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git checkout [branch_name]  //ブランチに移動
git commit -a -m "コメント"  //変更ファイルをコミット

git checkout master  //masterに移動
git merge [branch_name]  //差分をマージ
git push origin master  //ファイルの更新
</pre></div></div>

<h3>
<span id="マージを取り消す" class="fragment"></span><a href="#%E3%83%9E%E3%83%BC%E3%82%B8%E3%82%92%E5%8F%96%E3%82%8A%E6%B6%88%E3%81%99"><i class="fa fa-link"></i></a>マージを取り消す</h3>

<p>コンフリクトが発生して一旦戻したい場合</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git merge --abort
</pre></div></div>

<hr>

<h3>
<span id="差分を確認する" class="fragment"></span><a href="#%E5%B7%AE%E5%88%86%E3%82%92%E7%A2%BA%E8%AA%8D%E3%81%99%E3%82%8B"><i class="fa fa-link"></i></a>差分を確認する</h3>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git diff
git diff HEAD^ //最後のコミットからの差分を表示
git diff --name-only HEAD^ //差分ファイルを表示
git diff file1.txt file2.txt //特定フィイルの差分
git diff commit1 commit2 //コミットの差分
</pre></div></div>

<h3>
<span id="ログの表示" class="fragment"></span><a href="#%E3%83%AD%E3%82%B0%E3%81%AE%E8%A1%A8%E7%A4%BA"><i class="fa fa-link"></i></a>ログの表示</h3>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git log //コミットのログが見れる
git reflog //いろいろ見れる
git reflog origin/branch_name //pushのログが見れる
</pre></div></div>

<p>ログには色々なオプションがあるけど、おすすめは以下のコマンド。</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git log --graph --name-status --pretty=format:"%C(red)%h %C(green)%an %Creset%s %C(yellow)%d%Creset"
</pre></div></div>

<h3>
<span id="ファイルの名前変更" class="fragment"></span><a href="#%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%81%AE%E5%90%8D%E5%89%8D%E5%A4%89%E6%9B%B4"><i class="fa fa-link"></i></a>ファイルの名前変更</h3>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git mv [変更前のファイル名] [変更後のファイル名]
git commit -a -m "rename"
git push origin master
</pre></div></div>

<h3>
<span id="特定ファイルを特定のコミットに戻す" class="fragment"></span><a href="#%E7%89%B9%E5%AE%9A%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%92%E7%89%B9%E5%AE%9A%E3%81%AE%E3%82%B3%E3%83%9F%E3%83%83%E3%83%88%E3%81%AB%E6%88%BB%E3%81%99"><i class="fa fa-link"></i></a>特定ファイルを特定のコミットに戻す</h3>

<p>特定のコミットに戻してmasterに反映したい場合は以下のコマンドで。</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git checkout [commit_id] [file_name]  //特定ファイルの指定
git commit -a -m "return" //戻した内容をコミット
git push origin master //変更をプッシュ
</pre></div></div>

<h3>
<span id="今やってる作業を一時退避する" class="fragment"></span><a href="#%E4%BB%8A%E3%82%84%E3%81%A3%E3%81%A6%E3%82%8B%E4%BD%9C%E6%A5%AD%E3%82%92%E4%B8%80%E6%99%82%E9%80%80%E9%81%BF%E3%81%99%E3%82%8B"><i class="fa fa-link"></i></a>今やってる作業を一時退避する</h3>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git stash
git stash pop //戻す場合
git stash list //退避の一覧
git stash clear //退避の消去
</pre></div></div>

git tag -l 'v1.*' // パターンでタグを検索

git rm *  //全ファイルorディレクトリ
git commit -a -m "remove"  //削除をコミット
chgit push origin master  //削除を反映

git reset HEAD {file_name}
</pre></div></div>

<h3>
<span id="コンフリクトの解消" class="fragment"></span><a href="#%E3%82%B3%E3%83%B3%E3%83%95%E3%83%AA%E3%82%AF%E3%83%88%E3%81%AE%E8%A7%A3%E6%B6%88"><i class="fa fa-link"></i></a>コンフリクトの解消</h3>

<h4>
<span id="手動で解決する場合" class="fragment"></span><a href="#%E6%89%8B%E5%8B%95%E3%81%A7%E8%A7%A3%E6%B1%BA%E3%81%99%E3%82%8B%E5%A0%B4%E5%90%88"><i class="fa fa-link"></i></a>手動で解決する場合</h4>

<p>コンフリクトを解消しファイルを保存後、下記のコマンドを実行</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git add {file_name}
git commit {file_name} -m "コミットメッセージ"
</pre></div></div>

<h4>
<span id="自動で解決する場合" class="fragment"></span><a href="#%E8%87%AA%E5%8B%95%E3%81%A7%E8%A7%A3%E6%B1%BA%E3%81%99%E3%82%8B%E5%A0%B4%E5%90%88"><i class="fa fa-link"></i></a>自動で解決する場合</h4>

<p>現在のブランチを正とする</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git checkout --ours {fime_name}
</pre></div></div>

<p>マージで指定したブランチを正とする</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git checkout --theirs {fime_name}
</pre></div></div>

<p>また、mergetoolを使って解決することもできます。<br>
<a href="https://www.google.co.jp/url?sa=t&amp;rct=j&amp;q=&amp;esrc=s&amp;source=web&amp;cd=1&amp;cad=rja&amp;uact=8&amp;ved=0ahUKEwjdhfzf7PHKAhXMEpQKHbYpAtAQFggcMAA&amp;url=http%3A%2F%2Fqiita.com%2Fyuya_presto%2Fitems%2F5d99499cf96c0ebb09f8&amp;usg=AFQjCNFOOaTItltq5Fx2nFJDvodk3aAkvw&amp;sig2=tCg4ZXi3CVjmUG_Pf42LOA" rel="nofollow noopener" target="_blank">Gitコンフリクト解消ガイド（git mergetoolの使い方） - Qiita</a></p>

<h3>
<span id="圧縮ファイルの作成" class="fragment"></span><a href="#%E5%9C%A7%E7%B8%AE%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%81%AE%E4%BD%9C%E6%88%90"><i class="fa fa-link"></i></a>圧縮ファイルの作成</h3>

<p>詳しくは → <a href="http://qiita.com/kaminaly/items/28f9cb4e680deb700833" id="reference-0c67bda606c278ecee2e">gitで差分ファイルを抽出する</a></p>

<h4>
<span id="現在のリポジトリのzipファイルを作成" class="fragment"></span><a href="#%E7%8F%BE%E5%9C%A8%E3%81%AE%E3%83%AA%E3%83%9D%E3%82%B8%E3%83%88%E3%83%AA%E3%81%AEzip%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%92%E4%BD%9C%E6%88%90"><i class="fa fa-link"></i></a>現在のリポジトリのzipファイルを作成</h4>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git archive --format=zip HEAD -o ./hoge.zip
</pre></div></div>

<h4>
<span id="コミット間の差分を取得する" class="fragment"></span><a href="#%E3%82%B3%E3%83%9F%E3%83%83%E3%83%88%E9%96%93%E3%81%AE%E5%B7%AE%E5%88%86%E3%82%92%E5%8F%96%E5%BE%97%E3%81%99%E3%82%8B"><i class="fa fa-link"></i></a>コミット間の差分を取得する</h4>

<p>1つ前のコミットから差分を取得し、hoge.zipを作成する例</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git archive --format=zip --prefix=root/ HEAD `git diff --diff-filter=D --name-only HEAD HEAD^` -o hoge.zip
</pre></div></div>

<p>特定のコミット間の差分</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git archive --format=zip --prefix=root/ HEAD `git diff [old commit ID] [new commit ID] --name-only | git checkout-index --prefix=./diff_ --stdin` -o hoge.zip
</pre></div></div>

<p>なんかうまくいかない場合は、<code>zip</code>コマンドを試してみる。</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre> zip -r archive.zip `git diff --name-only HEAD [old commit ID]`
</pre></div></div>

<h4>
<span id="タグ間の差分を取得する" class="fragment"></span><a href="#%E3%82%BF%E3%82%B0%E9%96%93%E3%81%AE%E5%B7%AE%E5%88%86%E3%82%92%E5%8F%96%E5%BE%97%E3%81%99%E3%82%8B"><i class="fa fa-link"></i></a>タグ間の差分を取得する</h4>

<p>v1.0とv2.0間を差分ファイルを取得し、hoge.zipを作成する例</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git archive --format=zip --prefix=root/ v2.0 `git diff --name-only v1.0 v2.0` -o ./hoge.zip
</pre></div></div>

<p>.gitignoreファイルを使用する</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>#Directory
node_modules/
styleguide/

#一致するファイルすべて
*.txt

#aaa.cacheは除く
*.cache
!aaa.cache

</pre></div></div>

<h3>
<span id="git管理しているファイルをあえて無視する" class="fragment"></span><a href="#git%E7%AE%A1%E7%90%86%E3%81%97%E3%81%A6%E3%81%84%E3%82%8B%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%92%E3%81%82%E3%81%88%E3%81%A6%E7%84%A1%E8%A6%96%E3%81%99%E3%82%8B"><i class="fa fa-link"></i></a>git管理しているファイルをあえて無視する</h3>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git update-index --skip-worktree [ファイル名]
</pre></div></div>

<p>取り消す場合</p>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git update-index --no-skip-worktree [ファイル名]
</pre></div></div>

<p>詳しくは→ <a href="http://qiita.com/usamik26/items/56d0d3ba7a1300625f92" id="reference-63f2ec31c06780393f3d">既に git 管理しているファイルをあえて無視したい</a></p>

<h3>
<span id="gitignoreに記述されているファイルを管理対象から外す" class="fragment"></span><a href="#gitignore%E3%81%AB%E8%A8%98%E8%BF%B0%E3%81%95%E3%82%8C%E3%81%A6%E3%81%84%E3%82%8B%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%92%E7%AE%A1%E7%90%86%E5%AF%BE%E8%B1%A1%E3%81%8B%E3%82%89%E5%A4%96%E3%81%99"><i class="fa fa-link"></i></a>.gitignoreに記述されているファイルを管理対象から外す</h3>

<div class="code-frame" data-lang="text"><div class="highlight"><pre>git rm --cached `git ls-files --full-name -i --exclude-from=.gitignore`
</pre></div></div>
